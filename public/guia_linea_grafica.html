4. Márgenes y Espaciado

Márgenes Estrechos

Los márgenes deben ser mínimos para aprovechar al máximo el área disponible en pantalla. Esto mejora la densidad de información y reduce el desplazamiento innecesario.

Espaciado Recomendado:

  • Padding de contenedores: 0.5rem - 1rem (8px - 16px)
  • Márgenes entre secciones: 0.75rem - 1rem (12px - 16px)
  • Gap entre elementos: 0.5rem (8px)
  • Padding de botones: 1rem 1.5rem (16px vertical, 24px horizontal)

Evitar márgenes excesivos como 2rem o 3rem que desperdician espacio valioso en pantalla.

5. Iconos

Minimizar el Uso

Los iconos deben usarse con moderación. Solo cuando aporten valor real a la comprensión del contenido.

Iconos que NO Parecen Botones

Los iconos que NO son clickeables deben tener un estilo plano, sin fondo, sin borde, sin padding, y con opacidad reducida para que no parezcan botones.

Iconos Planos (Correcto):

Estos iconos son informativos, no clickeables. No tienen fondo ni borde.

Iconos que Parecen Botones (Incorrecto):

Estos iconos tienen fondo y padding, lo que los hace parecer botones clickeables cuando no lo son.

Reglas para Iconos

  • Iconos informativos: Sin fondo, sin borde, sin padding, opacidad 0.7, tamaño 1.2rem, color negro o blanco según el fondo
  • Iconos en botones: Color blanco, pueden tener fondo semitransparente solo dentro de botones reales
  • Usar Font Awesome: Siempre usar iconos de Font Awesome, nunca emojis

Indicadores y Badges

Los indicadores de estado (badges, etiquetas, estados) deben usar tonos pastel de los colores primarios, sin efecto bevel/relieve ni bordes pronunciados, para que no parezcan botones.

Indicadores con Colores Pastel (Correcto):

Completo Parcial Pendiente
Activo En Proceso

Estos indicadores usan tonos pastel sin box-shadow ni bordes, combinados con texto o iconos.

Indicadores que Parecen Botones (Incorrecto):

Completo

Este indicador tiene color primario intenso y efecto bevel, lo que lo hace parecer un botón clickeable.

Colores Pastel para Indicadores

Verde Pastel
#c8e6c9

Para estados positivos, completados, activos

Amarillo Pastel
#fff9c4

Para estados parciales, en proceso, advertencias

Rojo Pastel
#ffcdd2

Para estados pendientes, errores, atención requerida

Azul Pastel
#BBDEFB

Para estados informativos, en proceso, navegación

Reglas para Indicadores

  • Fondo pastel: Usar tonos pastel (#c8e6c9, #fff9c4, #ffcdd2, #BBDEFB) en lugar de colores primarios intensos
  • Sin bevel/relieve: No usar box-shadow ni efectos 3D
  • Sin bordes pronunciados: Solo border-radius suave (6px), sin bordes visibles
  • Texto negro: Siempre texto negro (#000000) sobre fondos pastel para legibilidad
  • Combinar con iconos: Los indicadores pueden incluir iconos de Font Awesome junto al texto
  • Padding moderado: Padding de 0.5rem 0.75rem para mantener proporción sin parecer botón

Recomendaciones

  • Uso moderado: Solo usar animaciones cuando sea necesario indicar interacción. No sobrecargar la interfaz
  • Icono recomendado: 1. fa-chevron-down es el más común y reconocible para despliegue
  • Animación recomendada: 1. icon-expand-bounce es sutil y efectiva
  • Tamaño: Usar 1.8rem o más para iconos en botones y acordeones (más notorio)
  • Color: Mantener opacidad 0.7 en fondos blancos, blanco en botones de color
  • Desactivar al abrir: Cuando el elemento esté desplegado, quitar la animación y rotar el icono 180°
  • Identificación: Las animaciones están numeradas del 1 al 4, y los iconos del 1 al 4 para fácil referencia

6. Botones

Estilo de Botones

Los botones deben tener un efecto de bevel/relieve pronunciado para que sea obvio que son clickeables.

Características de los Botones

  • Fondo sólido: Sin gradientes, usar colores planos (#0b7835, #fbdc02, #ea0e2a)
  • Texto blanco: Siempre texto blanco sobre fondos de color
  • Bevel/Relieve: Box-shadow con sombras internas y externas
  • Padding: 1rem 1.5rem (16px vertical, 24px horizontal)
  • Border-radius: 12px
  • Font-weight: 600 (Semi-Bold)

Código CSS para Botones

.button {
    background: #0b7835;
    color: white;
    padding: 1rem 1.5rem;
    border-radius: 12px;
    font-weight: 600;
    border: none;
    box-shadow: 0 3px 6px rgba(0, 0, 0, 0.25), 
                inset 0 2px 0 rgba(255, 255, 255, 0.25), 
                inset 0 -1px 0 rgba(0, 0, 0, 0.2);
}

8. Resumen de Reglas

HACER

  • Usar solo 3 colores primarios: Verde (#0b7835), Amarillo (#fbdc02), Rojo (#ea0e2a)
  • Fondos blancos (#ffffff) para contenedores de contenido
  • Textos negros (#000000) para máximo contraste
  • Fuente Inter en todos los elementos
  • Márgenes estrechos (0.5rem - 1rem) para aprovechar el área
  • Iconos planos sin fondo para elementos no clickeables
  • Indicadores con colores pastel (#c8e6c9, #fff9c4, #ffcdd2) sin bevel ni bordes
  • Botones con efecto bevel/relieve pronunciado
  • Secciones con fondo gris claro (#f5f5f5) y contenido blanco

NO HACER

  • Usar gradientes en fondos o botones
  • Textos grises sobre fondos blancos
  • Márgenes excesivos que desperdician espacio
  • Iconos con fondo que parezcan botones cuando no lo son
  • Indicadores con colores primarios intensos o efectos bevel/relieve
  • Usar emojis (solo iconos de Font Awesome)
  • Botones sin efecto bevel/relieve
  • Fondos de diferentes tonos de gris

Esta guía debe ser consultada antes de realizar cualquier cambio visual en el sistema.

12. Formularios

Los formularios deben ser claros, accesibles y seguir la línea gráfica establecida. Todos los campos deben tener etiquetas visibles, validación clara y feedback visual apropiado.

Tipos de Campos

El sistema soporta los siguientes tipos de campos de formulario:

  • Campo de texto: Para entrada de texto libre
  • Selección única: Radio buttons para opciones mutuamente excluyentes
  • Selección múltiple: Checkboxes para seleccionar múltiples opciones
  • Dropdown con buscador: Select con búsqueda dinámica y orden alfabético
  • Rango de fechas: Selector de fecha inicio y fin
  • Monto en dinero: Campo numérico con formato de moneda
  • Detalles/Descripción: Textarea para texto largo

Ejemplo Completo de Formulario

₡
Formato: ₡123.456.789

Características del Formulario

  • Etiquetas claras: Todas las etiquetas son visibles, con asterisco rojo para campos requeridos
  • Espaciado consistente: Gap de 1rem entre campos, 0.5rem entre label y campo
  • Bordes suaves: Border-radius de 8px para inputs, 12px para botones
  • Colores primarios: Verde para botón principal, gris para cancelar
  • Feedback visual: Hover en radio buttons, checkboxes y botones
  • Formato de moneda: Campo de monto con símbolo ₡ y formato de miles
  • Dropdown inteligente: Búsqueda dinámica con orden alfabético

Dropdown con Buscador - Funcionalidad

El dropdown con buscador permite:

  • Búsqueda en tiempo real mientras se escribe
  • Orden alfabético automático de las opciones
  • Mostrar todas las opciones cuando el campo está vacío
  • Selección por clic en la opción
  • Cierre automático al seleccionar o hacer clic fuera
